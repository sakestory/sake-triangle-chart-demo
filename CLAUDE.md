# 日本酒三角チャート診断アプリ — Claude作業ガイド

## このリポジトリ
- `index.html` … 診断アプリ本体（単一ファイル。データもロジックも全部この中）
- `cat_01.png`〜`cat_27.png` … 27タイプの酒猫画像
- `index_.html` … 旧版の控え。**触らない**
- デプロイ：main に push されると GitHub Actions（deploy-lolipop.yml）が
  `index.html`＋`cat_*.png` を SFTP で https://shindan.sake-story.com/ へ自動アップロード

## 正本（元栓）のルール
- 銘柄座標の唯一の正本＝ **hashino_os リポジトリ内の「銘柄座標マスター vX.X」Numbersファイル**（最新版を使う。旧版は old/ にある）
- Excel/Numbers＝橋野さんの編集の入口。JSON・アプリ側は「写し」。**アプリ側の座標を勝手に直さない**（次の詰め替えで消える）
- 分類「店データ」の20銘柄は聖域（橋野さんの確定値）。指示なしに変更禁止
- 入手マークは「橋野さん修正欄」が入っていればそちらが正、空欄ならマーク列のまま

## 「更新したよアップして」と言われたら（定型ワークフロー）
1. hashino_os から最新の銘柄座標マスター（.numbers）を取得
   （セッションに hashino_os が無い場合はアップロードを依頼。python の `numbers-parser` で読める）
2. マスターから index.html の3つのデータを詰め替える：
   - `STORE_SAKE` … 店内◯の銘柄のみ・マスターの並び順・座標は整数
   - `NATL_SAKE` … 店内◯**以外**の全銘柄・マスターの並び順・mark は修正欄優先
   - `CHART_HIDDEN` … チャート表示チェックが**外れている**銘柄名の配列
     （チャート・シェア画像から除外される。診断27タイプ判定には影響しない）
3. **銘柄名の例外**：マスターの「会津娘無為信」は店内表示では「無為信」にする
   （全国側の「会津娘」との重複回避。2026-07-15決定）
4. 一致検証（必須）：書き換え後の index.html からデータを再抽出し、
   本数・名前・座標・マーク・非表示リストをマスターと機械照合して完全一致を確認
5. JS構文チェック（`<script>` を抜き出して `node --check`）
6. フッターのバージョン表記（2箇所、`v11.x.y`）をパッチ番号+1
7. コミット → `git push` → PR作成 → **マージまで実行**（橋野さん承認済みの運用）
8. デプロイworkflowの成功と、ログ内の HTTP 200 チェックを確認して報告

## その他の決まりごと
- 店内おすすめ欄に NATL_SAKE の銘柄を混ぜない（混入事故防止）
- 文言（タイプ文・味わい文など）の正本は shindan_logic_master のマスターMD。座標とは別管理
- 診断ログは Google Apps Script → スプレッドシートに記録（`?test=1`・localhost は除外）
- コミットメッセージは日本語で「vX.Y.Z 内容」の形式
